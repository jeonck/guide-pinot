3. 데이터 모델링

3.1 테이블 유형

Pinot에서는 주로 세 가지 유형의 테이블을 사용합니다:

  • 오프라인 테이블 (Offline Table): 배치 데이터 처리용, HDFS, S3 등에서 데이터 로드, 불변 세그먼트 사용
  • 실시간 테이블 (Realtime Table): 실시간 데이터 스트리밍용, Kafka와 같은 메시지 큐 사용, 실시간 데이터를 작은 세그먼트로 저장
  • 하이브리드 테이블 (Hybrid Table): 오프라인과 실시간 테이블의 기능 결합, 실시간 데이터와 과거 데이터를 함께 분석

3.2 스키마 정의

Pinot 스키마는 다음과 같은 주요 구성요소를 가집니다:

  • 차원 컬럼 (Dimension Column): 필터링, 그룹화, 정렬에 사용 (예: 사용자 ID, 카테고리, 지역)
  • 메트릭 컬럼 (Metric Column): 집계 연산에 사용되는 수치형 데이터 (예: 판매량, 가격, 클릭 수)
  • 타임 컬럼 (Time Column): 시간 데이터 저장 및 시간 기반 쿼리 수행, 타임스탬프 데이터 타입 사용

스키마 예시:

{
  "schemaName": "exampleSchema",
  "dimensionFieldSpecs": [
    {
      "name": "userId",
      "dataType": "STRING",
      "singleValueField": true
    },
    {
      "name": "category",
      "dataType": "STRING",
      "singleValueField": true
    }
  ],
  "metricFieldSpecs": [
    {
      "name": "price",
      "dataType": "DOUBLE",
      "singleValueField": true
    },
    {
      "name": "quantity",
      "dataType": "INT",
      "singleValueField": true
    }
  ],
  "timeFieldSpec": {
    "name": "eventTime",
    "dataType": "TIMESTAMP",
    "format": "1:ISO",
    "timeType": "DATETIME"
  }
}

3.3 데이터 타입

Pinot은 다양한 데이터 타입을 지원합니다:

  • 기본 데이터 타입: STRING, INT, LONG, FLOAT, DOUBLE, BOOLEAN, TIMESTAMP
  • 복합 데이터 타입: ARRAY, MAP
  • 특수 데이터 타입: BYTES, DECIMAL
  • 단일 값 및 다중 값 필드: 단일 값 필드는 레코드당 하나의 값, 다중 값 필드는 여러 값 저장 가능
