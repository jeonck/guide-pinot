2. Apache Pinot 아키텍처

2.1 주요 컴포넌트

Pinot의 아키텍처는 다음과 같은 주요 컴포넌트로 구성됩니다:

  • Pinot Controller: 클러스터 관리, 세그먼트 관리, 메타데이터 관리
  • Pinot Server: 데이터 저장 및 쿼리 처리
  • Pinot Broker: 쿼리 라우팅 및 결과 병합
  • Zookeeper: 클러스터 조정

2.2 데이터 흐름 아키텍처

Pinot의 데이터 흐름은 데이터 소스에서 시각화까지 다음과 같은 단계를 거칩니다:

+---------------------+       +---------------------+       +---------------------+
|  Data Sources       | ----> |  Data Ingestion     | ----> |  Pinot Cluster      |
|(e.g., Logs, Sensors)|       | (e.g., Kafka, Flume)|       |                     |
+---------------------+       +---------------------+       +---------------------+
                                                      |
                                                      v
                                             +---------------------+
                                             |  Pinot Query Layer  |
                                             | (e.g., Pinot Broker)|
                                             +---------------------+
                                                      |
                                                      v
                                             +---------------------+
                                             |  Data Visualization |
                                             |(e.g.,Grafana,Kibana)|
                                             +---------------------+

이 아키텍처는 데이터 소스(로그, 센서 등)에서 데이터를 수집하여 Kafka나 Flume 같은 도구로 데이터를 수집하고, Pinot 클러스터에서 처리한 후, Broker를 통해 쿼리를 처리하고 결과를 시각화 도구로 전달합니다.
